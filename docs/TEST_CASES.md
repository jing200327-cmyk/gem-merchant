# 规则与 AI 锁定测试

运行 `npm test` 执行全部测试。

## 规则

- 宝石上限、强制归还、同色库存门槛与阶段操作隔离。
- 官方支付计算：白 1、蓝 1、红 1、黄金 2，黄金剩余 0。
- 单贵族自动获得，多贵族选择一个，宝石不计贵族条件。
- 完成本轮后终局；同分时购买发展卡较少者获胜，仍相同则并列。
- Action/Event、快照、不变性、导入校验和日志回放。

## AI

- 随机 AI 的选择合法且固定种子下确定。
- 贪心 AI 优先高声望购买；规划 AI 会围绕高价值目标行动。
- AI 可处理 `AWAITING_ACTION`、`RETURNING_GEMS`、`CHOOSING_NOBLE`、`TURN_TRANSITION`、`GAME_OVER`。
- AI 可购买预留卡、从牌堆顶盲抽预留，并在结束后停止。
- 空宝石池与三个预留位组合不会使 AI 死锁。
- 四 AI 对局可结束并由动作日志完整复盘。

## 压力与浏览器

- `npm run simulate:10000`：10,000 局，非法动作 0、无合法行动 0、无限局 0。
- 真人浏览器流程：设置 → 拿取宝石 → 回合遮罩 → 日志。
- 混合浏览器流程：真人行动 → AI 确认回合 → AI 执行合法主要行动。
